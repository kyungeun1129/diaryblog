# Project_3
종류	교육노동부 주관 정부 교육 과정 (KG ITBANK)
프로젝트명	나만의 블로그 만들기
개발기간	2024.11.11 ~ 2024.11.20
참여인원	3명
담당업무	아이디어 제안, 기획, 메인 페이지 설계, 일기 게시판 설계 및 제작
개발환경	Windows
사용도구	Eclipse, Oracle
사용기술	Java

개요

‘나만의 블로그 만들기’ 프로젝트는 사용자가 자신만의 독특한 공간을 창조하고, 다양한 주제에 대한 생각을 자유롭게 표현하고 공유할 수 있도록 설계되었습니다.

- 회원가입, 인증, 로그인과 person 데이터베이스 연동
- 메인 게시판 : 닉네임 연동 이웃 설정, 랜덤 오늘의 운세, 음악 기능, 달력 기능
- 맛집 게시판 : 맛집 리스트, 맛집 수정 기능, URL 연결 지도, foodList 데이터베이스 연동
- 일기 게시판 : 일기 쓰기, 일기 저장, 일기 확인 및 삭제
- cardLayout을 통해 전체적으로 자연스러운 화면 전환 구성

기획 의도(동기)

이 프로젝트는 개인의 취향과 개성을 반영할 수 있는 맞춤형 기능들이 추가되어, 사용자에게 단순화된 블로그 프로그램을 사용하는 것이 아니라 정말로 ‘나만의’ 블로그를 만들어볼 수 있는 기회를 제공합니다.

프로젝트는 크게 세 가지 주요 게시판으로 나누어집니다. 첫 번째인 메인 게시판에서는 사용자가 자신의 개성과 스타일을 드러낼 수 있도록 다양한 멀티미디어 콘텐츠를 추가하였습니다. 두 번째, 맛집 게시판은 사용자가 음식에 대한 정보를 본인이 개인적으로 수정할 수 있도록 구성되었고, 그 결과를 글과 이미지를 통해 직관적으로 확인할 수 있는 환경을 제공합니다.
마지막으로 일기 게시판은 사용자가 개인적인 생각과 감정을 자유롭게 기록하고, 다른 사람들과 공유할 수 있는 공간입니다. 이 게시판에서는 사용자가 자신만의 일기를 작성하거나 이미 게시된 일기들을 삭제할 수 있습니다. 게시된 일기들은 로그인한 다른 사용자들과 공유되어 소통의 기회를 만들어줍니다.

목표 및 설계
이 프로젝트의 핵심 목표는 사용자가 자신의 일상, 취향, 경험을 기록하고 다른 사용자들과 자유롭게 공유할 수 있는 환경을 제공하는 것입니다. 누구나 손쉽게 사용할 수 있도록 직관적인 화면 전환과 UI/UX 설계에도 중점을 두었습니다. 
[일기 게시판]
저는 일기 게시판을 주로 담당하였으며, 목표는 사용자가 개인적인 생각과 감정을 기록하고 다른 사용자들과 자유롭게 공유할 수 있는 환경을 만드는 것입니다. 이를 위해 사용자가 일기를 작성하고 저장할 수 있도록 직관적인 환경을 구성하고 일기 게시물의 삭제 및 수정 기능을 원활하게 구현하는 것에 중점을 두었습니다.

후기

이번 프로젝트를 통해 Java의 파일 시스템과 연동되는 방식을 새롭게 배울 수 있었습니다. 파일을 저장하고 읽어오는 과정에서 FileOutputStream, OutputStreamWriter, FileinputStream, InputStreamReader,
BufferReader 등을 활용하여 데이터를 파일로 저장하고 다시 불러오는 흐름을 익히며 수업 중 배웠던 기초를 실제로 적용해볼 수 있었습니다.

화면 간 전환에서 정보가 제대로 유지되지 않아 어려움을 겪었지만, CardLayout을 활용하여 그 문제를 해결하면서 화면 간 자연스럽고 유연한 전환을 구현할 수 있었습니다. 버튼의 위치와 기능을 사용자 입장에서 고민하며 UI를 구성한 경험은, 사용자 경험(UX)을 고려하여 Event와 인터페이스를 설계하는 것이 얼마나 중요한지 깨닫게 해주었습니다. 특히 각 화면 간의 흐름을 자연스럽게 연결하는 과정에서 UI/UX 설계 역량을 키울 수 있었습니다.

아쉬운 점도 있었습니다. 글을 작성한 사용자만 글을 삭제할 수 있도록 제한하는 기능과 댓글 기능은 시간 부족으로 완성하지 못했습니다. 이를 보완하기 위해서는 파일을 저장할 때 제목과 작성일뿐만 아니라 작성자의 닉네임 정보도 함께 저장하고, 메인 화면과 연동하여 해당 정보로 권한을 확인하면 해결할 수 있을 것이라 생각합니다. 이렇게 하면 작성자만 글을 삭제할 수 있도록 제한할 수 있고, 다른 사용자는 댓글을 작성하는 방식으로 확장할 수 있을 것입니다. 이후 프로젝트를 혼자 다시 구현해보면서 이러한 기능을 추가하는 것이 좋은 연습이 될 것이라고 생각합니다.

팀워크 면에서도 많은 성장을 느꼈습니다. 프로젝트 초반에는 서로 어색한 분위기였지만, 주도적으로 자기소개를 시작하고 분위기를 부드럽게 만들기 위해 노력했습니다. 이러한 작은 시도들이 팀원들과의 친밀감을 형성하는 데 큰 도움이 되었고, 자연스럽게 의사소통이 활발해졌습니다. 아이디어를 구상하는 초기 단계에서는 막막한 느낌도 있었지만, 아이디어는 많을수록 좋다는 생각으로 머릿속에 떠오르는 모든 아이디어를 필기하며 적극적으로 공유했습니다. 결과적으로 다양한 아이디어들이 모여 '나만의 블로그 만들기'라는 주제로 이어질 수 있었습니다.

진행 과정에서는 정기적인 팀 회의를 통해 진척 상황을 공유하였고, 회의에 적극적으로 참여하기 위해 작업한 내용을 따로 정리해 가며 현재 상태와 방향성을 명확하게 전달하고자 노력했습니다. 혼자서 해결하기 어려운 문제는 팀원에게 주저하지 않고 도움을 요청했고, 점차 팀 내에서 다른 사람을 도울 수 있는 위치까지 성장하게 되었습니다. 이런 협업 과정 속에서 유연한 성격과 적극적인 소통 태도가 팀워크를 높이는 데 큰 역할을 했다고 느꼈습니다. 이번 프로젝트는 기술적인 성장뿐 아니라 협업 능력, 문제 해결력, 커뮤니케이션 역량까지 고루 향상시킬 수 있었던 값진 경험이었습니다.
